═══════════════════════════════════════════════════════════════════
STEP 4 — CONNECT OUTLOOK CALENDAR (optional but recommended)
═══════════════════════════════════════════════════════════════════

Jarvis uses Microsoft Graph API to read your Outlook calendar.
You need to give it a temporary access token.

EASY METHOD (no coding, works for personal testing):

1. Go to: https://developer.microsoft.com/en-us/graph/graph-explorer
2. Click "Sign in" (top right) — log in with your Microsoft/Outlook account
3. You'll see a green "Signed in as..." banner
4. In the left panel, click "Access token" tab
5. Copy the entire token (it's a long string starting with "eyJ...")
6. Paste it into Jarvis Settings → Microsoft Graph Bearer Token

NOTE: These tokens expire after 60-90 minutes.
For a permanent solution, contact Alfred to set up an Azure app registration
(this requires some IT setup but gives a permanent token via OAuth).

STEP 5 — CONFIGURE MORNING BRIEFING
═══════════════════════════════════════════════════════════════════

In Jarvis Settings:
1. Toggle "Enable morning briefing" ON
2. Set your wake-up time using the hour/minute pickers
3. Tap Save Configuration
4. Close the app and reopen it (so the alarm registers)

At your set time each morning, Jarvis will automatically speak:
  "Good morning, Alfred. In Johannesburg right now, it is 14 degrees..."
  followed by your full calendar for the day.

Your phone does NOT need to be unlocked — it works from the lock screen.

PERMISSIONS YOU MUST GRANT ON FIRST RUN
═══════════════════════════════════════════════════════════════════

When you first install and launch the app, you'll be prompted for:

1. Microphone — REQUIRED. Jarvis cannot hear you without this.
2. Phone — for direct calling (without this, it opens the dialler instead).
3. Contacts — optional but lets Jarvis resolve names to numbers.
4. Send SMS — for SMS actions.
5. Calendar — for the morning briefing (reads your device calendar as backup).
6. Notifications — for the persistent status notification.

If you accidentally denied any permission:
  Settings → Apps → J.A.R.V.I.S → Permissions → enable each one


═══════════════════════════════════════════════════════════════════
TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════

Problem: "I couldn't find that app on your phone"
Fix: The app may not be installed, or Jarvis doesn't know its package name yet.
     Say the exact app name. If it still fails, Claude.ai can add the package
     name to the APP_PACKAGES map in ActionExecutor.kt.

Problem: Morning briefing doesn't fire
Fix: 1. Check battery optimisation is set to "No restrictions" for Jarvis.
     2. Toggle the morning briefing OFF and ON in settings, then restart Jarvis.
     3. Nothing OS sometimes kills alarms — "No restrictions" is critical.

Problem: Outlook calendar shows "could not access"
Fix: Your Microsoft Graph token has expired (they last 60-90 minutes).
     Go to graph-explorer, sign in again, copy a fresh token, paste in settings.

Problem: Call just opens the dialler
Fix: Settings → Apps → J.A.R.V.I.S → Permissions → Phone → Allow

Problem: WhatsApp message opens but the contact is wrong
Fix: Make sure you say the number in SA format: "zero eight two one two three..."
     Jarvis automatically converts 0xx to +27xx for WhatsApp.
